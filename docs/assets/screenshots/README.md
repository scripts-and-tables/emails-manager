# Screenshots

Drop PNGs here under the filenames referenced by `docs/screenshots.html`.
Until they're added, the gallery renders with broken-image icons and captions —
which is intentional, so you know which shots are still missing.

## Expected filenames

| Filename | What it shows |
| --- | --- |
| `01-landing.png` | The public landing page (logged-out view) |
| `02-login.png` | Login form |
| `03-otp.png` | 6-digit one-time-code entry |
| `04-home.png` | Signed-in dashboard with the three tiles |
| `05-accounts.png` | Accounts list with status dots and per-row actions |
| `06-bulk-add.png` | Bulk CSV import flow |
| `07-inbox.png` | Inbox view with filters (window / folder / account / group) |
| `08-email-detail.png` | Single email message body view |
| `09-guide.png` | The in-app Guide page |

## Sizing notes

- Capture at 2× device pixel ratio if you can (Retina / HiDPI); browser will
  scale them down crisply.
- Target longest edge around 1600–2000 px; PNGs over ~600 KB should be
  run through a lossless optimiser (TinyPNG, oxipng, ImageOptim).
- Keep the same browser chrome cropping across all shots for a tidy gallery.
